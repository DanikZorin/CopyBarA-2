# Дополнительная спецификация

## 1. Надёжность

| Требование | Описание |
|------------|----------|
| Обработка ошибок | Система корректно обрабатывает неверный ввод: неверный логин/пароль (4 попытки), неверный формат даты (исключение), пустые команды |
| Устойчивость к сбоям | При ошибке SQLite выводится сообщение `sqlite3_prepare_v2 ERROR!` с текстом ошибки |
| Валидация | Даты проверяются на корректность (год 0-9999, месяц 0-12, день 0-31) |

## 2. Удобство использования

| Требование | Описание |
|------------|----------|
| Интуитивность | Все команды начинаются с `-` (например, `-help`, `-tables`, `-sql`) |
| Подсказки | Команда `-help` выводит полный список доступных команд |
| Обратная связь | После каждого действия пользователь видит результат или сообщение `No Data!` |

## 3. Производительность

| Требование | Описание |
|------------|----------|
| Время отклика | SELECT-запросы выполняются не более 1 секунды |
| Запуск | Время запуска приложения не более 2 секунд |
| Аутентификация | Проверка логина/пароля выполняется мгновенно |

## 4. Понятность

| Требование | Описание |
|------------|----------|
| Модульность | Код разделён на модули: Server (обработка команд), Authenticator (вход), Date (работа с датами) |
| Именование | Функции названы осмысленно: `task1()`, `problem5()`, `masterStatUpdate()` |

## 5. Безопасность

| Требование | Описание |
|------------|----------|
| Аутентификация | Доступ к системе только после ввода логина и пароля |
| Ограничение попыток | Максимум 4 неудачных попытки входа |
| Защита от SQL-инъекций | Используются подготовленные выражения (`sqlite3_prepare_v2` с плейсхолдерами `?`) |

## 6. Технологические ограничения

| Ограничение | Значение |
|-------------|----------|
| Язык программирования | C++17 |
| Система управления базами данных | SQLite 3.35+ |
| Сборка | Makefile (g++) |
| CI/CD | GitHub Actions |
| Документация | Markdown + GitHub Pages |
| Платформа | macOS |
