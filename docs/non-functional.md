# Дополнительная спецификация

## 1. Надёжность

| Требование | Описание |
|------------|----------|
| Обработка ошибок | Система корректно обрабатывает неверный ввод пользователя (неверный формат даты, пустые поля, отрицательные числа) |
| Устойчивость к сбоям | При потере соединения с базой данных система выводит понятное сообщение и не теряет данные |
| Валидация | Все внешние ключи проверяются при вставке/обновлении данных |

## 2. Удобство использования

| Требование | Описание |
|------------|----------|
| Интуитивность | Консольное меню содержит понятные пункты с нумерацией |
| Подсказки | При вводе данных выводятся подсказки о формате (например, "ГГГГ-ММ-ДД") |
| Обратная связь | После каждого действия пользователь видит сообщение об успехе или ошибке |

## 3. Производительность

| Требование | Описание |
|------------|----------|
| Время отклика | SELECT-запросы выполняются не более 1 секунды |
| Запуск | Время запуска приложения не более 2 секунд |

## 4. Понятность

| Требование | Описание |
|------------|----------|
| Модульность | Код разделён на логические модули: модели (h), сервисы (cpp) |
| Именование | Переменные и функции названы осмысленно (на английском или транслите) |

## 5. Безопасность

| Требование | Описание |
|------------|----------|
| Аутентификация | Доступ к системе возможен только после ввода логина и пароля |
| Разграничение прав | Администратор имеет полный доступ, мастер — только просмотр своих работ |

## 6. Технологические ограничения

| Ограничение | Значение |
|-------------|----------|
| Язык программирования | C++17 |
| Система управления базами данных | SQLite 3.35+ |
| Сборка | Makefile или CMake |
| Тестирование | Google Test или CUnit |
| CI/CD | GitHub Actions |
| Документация | Markdown (Wiki + GitHub Pages) |
